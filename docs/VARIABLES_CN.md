# 路径模板变量使用手册

 EzImage 允许你通过灵活的变量自定义图片的存储路径和文件名。你可以在配置项 `ezimage.pathTemplate` 中进行设置。

## 1. 默认设置
默认值为：`{yyyy}/{MM}/{timestamp}-{random}.{ext}`
示例输出：`2024/03/1710651200000-a2b3c4d5.webp`

---

## 2. 可用变量列表

| 变量 | 说明 | 示例 (2024-03-01 14:30:00) |
| :--- | :--- | :--- |
| **`{yyyy}`** | 4 位年份 | `2024` |
| **`{yy}`** | 2 位年份 | `24` |
| **`{MM}`** | 2 位月份 | `03` |
| **`{dd}`** | 2 位日期 | `01` |
| **`{HH}`** | 2 位小时 (24小时制) | `14` |
| **`{mm}`** | 2 位分钟 | `30` |
| **`{timestamp}`** | 13 位毫秒级时间戳 | `1710651200000` |
| **`{random}`** | 8 位随机字符串 | `f9a2b8c5` |
| **`{filename}`** | 原始文件名 (不带扩展名) | `screenshot` |
| **`{ext}`** | 文件扩展名 | `png` (如果是自动转 WebP 则为 `webp`) |

---

## 3. 进阶使用技巧

### 场景 A：按月份分类，保持原始文件名
模板：`blog/{yyyy}-{MM}/{filename}.{ext}`
输出：`blog/2024-03/screenshot.webp`

### 场景 B：防止重名 (推荐)
为了避免同名文件在云端被覆盖，建议至少包含 `{random}` 或 `{timestamp}`。
模板：`assets/{yyyy}/{filename}-{random}.{ext}`

### 场景 C：纯时间轴风格
模板：`archives/{yyyy}/{MM}/{dd}/{timestamp}.{ext}`

---

## 4. 特别说明：自动转 WebP
如果插件开启了 `ezimage.compress` 功能，插件会将大部分图片格式自动转换为 **WebP**。此时变量 `{ext}` 将会自动变为 `webp`，无论你原始文件是什么格式。
