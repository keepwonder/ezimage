# Cloudflare R2 配置指南

本指南将手把手教你如何从零开始配置 Cloudflare R2 并将其集成到 EzImage 插件中。

## 1. 什么是 Cloudflare R2？
Cloudflare R2 是一种对象存储服务，它最迷人的地方在于 **0 流量分发费 (Zero Egress Fees)**。对于图床场景，这意味着无论你的图片被访问多少次，你都不需要支付流量费。

### 费用说明
- **存储**: 前 10GB 免费。
- **操作**: 每月前 100 万次 A 类操作（写）和 1000 万次 B 类操作（读）免费。
- **流量**: **完全免费**。

---

## 2. 获取 API 凭证 (Credentials)

在使用插件前，你需要获取 `Account ID`、`Access Key ID` 和 `Secret Access Key`。

1.  登录 [Cloudflare 控制台](https://dash.cloudflare.com/)。
2.  在左侧菜单选择 **R2**。
3.  在页面右侧找到并点击 **Manage R2 API Tokens**。
4.  点击 **Create API token**：
    *   **Token name**: 填写 `EzImage-Token`。
    *   **Permissions**: 选择 **Admin Read & Write** (或自定义为仅针对特定 Bucket 的读写)。
    *   **TTL**: 建议选择 **Forever**。
5.  点击 **Create API Token** 后，你将看到以下信息（**请务必立即保存，离开后将无法再次查看**）：
    *   **Access Key ID**
    *   **Secret Access Key**
    *   **Account ID** (在页面的 API 终结点 URL 中也可以找到，通常是 `https://<AccountID>.r2.cloudflarestorage.com`)

---

## 3. 创建与配置存储桶 (Bucket)

1.  回到 R2 概览页面，点击 **Create bucket**。
2.  输入名称（例如 `my-images`），点击 **Create**。
3.  **开启公开访问 (Public Access)**：
    *   进到刚才创建的 Bucket，点击 **Settings** 选项卡。
    *   找到 **Public Access** 区域。
    *   **方式 A (推荐)**: 绑定你的自定义域名（Custom Domain），例如 `img.yourdomain.com`。
    *   **方式 B**: 开启允许通过 R2.dev 访问（适合测试，但有速度限制且不稳定）。

---

## 4. 在 EzImage 中填入配置

1.  在 VS Code 中按下 `Cmd+Shift+P`。
2.  输入并选择 `EzImage: Configure Settings`。
3.  填入对应的参数：
    - **Provider**: `r2`
    - **Account ID**: 填入步骤 2 获取的 ID。
    - **Access Key ID**: 填入步骤 2 获取的 Access Key。
    - **Secret Access Key**: 填入步骤 2 获取的 Secret Key。
    - **Bucket Name**: 填入步骤 3 创建的名称。
    - **Public URL**: 填入步骤 3 绑定的自定义域名（例如 `https://img.yourdomain.com`），注意末尾不要带斜杠。

---

## 5. 常见问题排查

-   **上传 403 错误**: 请检查 API Token 的权限是否授予了读写权限，以及 `Account ID` 是否填写正确。
-   **图片无法显示**: 请确保在 Bucket 的 Settings 中已经开启了 **Public Access** 并正确绑定了域名。
-   **域名未生效**: 如果是新绑定的域名，可能需要几分钟的时间进行 DNS 同步。
