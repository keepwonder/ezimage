# VS Code Marketplace 上架指南

将 EzImage 上架到官方市场，可以让更多用户通过搜索直接安装，并享受自动更新的便利。以下是详细的上架步骤：

## 1. 准备工作

在开始之前，请确保你已经安装了 `vsce`（VS Code Extension Manager）：

```bash
npm install -g @vscode/vsce
```

---

## 2. 创建 Microsoft 账户与 Azure DevOps 组织

VS Code 市场的认证是基于 Azure DevOps 的令牌（PAT）的。

1.  **创建 Microsoft 账号**: 如果没有，去 [outlook.com](https://outlook.com) 注册一个。
2.  **登录 Azure DevOps**: 访问 **[https://aex.dev.azure.com/me](https://aex.dev.azure.com/me)**。
    
    ![ezimage-1772428036126.png](https://images.flashnote.top/2026/03/1772428036483-j3erp8xj.png)
    *   **⚠️ 防走丢提示**: 如果点击主页的 "Sign in" 跳转到了 Azure Portal，请直接手动在浏览器输入上面的 `aex.dev.azure.com/me`。这是管理组织的**直达通道**。
3.  **点击 "New organization"**: 
    *   通常在登录后的左下角。
    *   如果系统提示选择地区，建议选 `Central US` 或 `East Asia`。
    *   **注意**: 即使带有 "Azure" 字样，此步骤也是**完全免费**的，不需要绑定信用卡或购买订阅。
    ![ezimage-1772428188590.png](https://images.flashnote.top/2026/03/1772428188985-ye6tqzly.png)

    需要有一个订阅否则点击 continue 没有响应

    ![ezimage-1772428883381.png](https://images.flashnote.top/2026/03/1772428883561-uvq0ekf1.png)
4.  **完成验证**: 名字可以随便起（例如 `kiang-dev`），完成后即可进入组织后台。

---

## 3. 获取个人访问令牌 (Personal Access Token, PAT)

这是上架过程中最关键的一步，用于授权 `vsce` 命令行工具代表你进行操作。

1.  在 Azure DevOps 右上角点击 **User settings**（人像图标左侧的一个小图标）-> **Personal access tokens**。
2.  点击 **New Token**：
    *   **Name**: 填写 `EzImage-Publish`。
    *   **Organization**: 选择 **All accessible organizations**。
    *   **Scopes**: 选择 **Custom defined**。
    *   点击下方的 **Show all scopes**，找到 **Marketplace** 项：
        *   勾选 **Acquire** 和 **Manage**。
3.  点击 **Create**。

![ezimage-1772428961523.png](https://images.flashnote.top/2026/03/1772428961905-r182b0fc.png)
4.  **重要**: 复制并保存这个生成的令牌（Token），它只会出现这一次。

---

## 4. 创建发布者 (Publisher)

发布者是你在 VS Code 市场上的身份标识。

1.  访问 [VS Code Marketplace 管理页面](https://marketplace.visualstudio.com/manage)。
2.  点击 **Create Publisher**：
    *   **Name**: 填写你的发布者名称（例如 `kiang`）。这必须与 `package.json` 中的 `"publisher": "kiang"` 字段完全一致！
    *   **ID**: 唯一的标识符。
3.  填写完基本信息后保存。

---

## 5. 命令行登录与发布

回到本地终端，进入项目根目录：

### 5.1 登录
```bash
vsce login <你的发布者名称>
# 随后会提示输入你刚才保存的 PAT 令牌
```

### 5.2 验证发布信息
发布前请检查 `package.json`：
- `publisher`: 必须与市场创建的一致。
- `icon`: 必须是 `icon.png` 的本地路径。
- `repository`: 必须指向 GitHub 仓库。

### 5.3 正式发布
```bash
vsce publish
```
发布成功后，插件通常需要几分钟进行病毒扫描和初步审核，之后就可以在市场搜索到了。

---

## 6. 后续自动化上架 (CI/CD)

一旦你完成了手动发布的第一步，我们可以升级 GitHub Actions 脚本，让你每次推送 Tag 时不仅创建 Release，还自动把新版本推送到 Marketplace：

1.  在 GitHub 仓库中，进入 **Settings** -> **Secrets and variables** -> **Actions**。
2.  添加一个新的 Secret：`VSCE_PAT`，值就是你的 Azure DevOps 令牌。
3.  更新 `.github/workflows/release.yml`，增加一步 `vsce publish` 即可。
